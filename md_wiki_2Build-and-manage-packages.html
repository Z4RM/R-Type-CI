Build and manage packages

Prerequesties

To be able to build and/or manage packages of the R-Type project, you need to have CMake and Conan installed on your system.

CMake

Download CMake from the official website and install it. The project needs at least CMake 3.29, but the newer the version the better (unless the version is incompatible with the code, in this case, take an older version, or just the 3.29).

Conan

Download and install Conan using this command:

pip install conan

If you do not have or do not want to use Pip, you can use another method, listed on the official website.

Warning
Don't forget to add the location of your Conan executable in your PATH (by default, it is installed in ~/.local/bin/ on Linux).

Configure your code editor/IDE with CMake and Conan

Jetbrains (CLion)

  1. When you open for the first time the project in your IDE, a dialog asking you to configure CMake will open. Click on "OK" (you can first configure the profile if you want).
  2. An error saying that CMake cannot find packages will be displayed, but do not worry, it is normal.
  3. Install the Conan plugin (Managing plugins in CLion).
  4. Click on the Conan logo in the left sidebar (if there is none, click on the three horizontal dots in this same sidebar).
  5. Click on the three vertical dots (or gear) on the left ("Configure Conan").
  6. Select "Use Conan installed in the system" and let the three others as is (normally selected).
  7. Click on "OK". Conan will configure all needed so your IDE can build and you can execute the R-Type.
Remarks
If it does not worked, try to manually set the path location of you Conan executable in the "Conan executable" field instead of selecting "Use Conan installed in the system".

And that's it, build is now configured on your IDE!

This will also have added r-type_server and r-type_client to the configurations. You can so build, run and debug them from your IDE.

Manage packages

Add a package

To add a package, nothing simpler:

  1. Go to the ConanCenter.
  2. Search for your package.
  3. If the one you search for is in the list, so you can install and use it. However, it will not be possible.\ Click on the package you want.
  4. Add <package_name>/<version> in the conanfile.txt, under the [requires] section.
  5. If something in the conanfile.txt from the website is not present in the project one, add it.
  6. Add find_package(<package_name> REQUIRED) and target_link_libraries(<target> PRIVATE <target_name>) (see Targets on the website) in the CMakeLists.txt
  7. Add target_link_libraries(<target> PRIVATE <target_name>) (also see Targets on the website) in the CMakeLists.txt.

And that's it, Conan will download the package the next time you build, and you can now use it in your code.

Update a package

To update a package, just change its version in the conanfile.txt be careful that it does not break any feature.

Remove a package

  1. Remove all code related to this package.
  2. Remove it from the CMakeLists.txt.
  3. Remove it from the conanfile.txt
